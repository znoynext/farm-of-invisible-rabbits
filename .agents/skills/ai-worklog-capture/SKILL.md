# AI Worklog Capture

1. Прочитать `../../../docs/ai-worklog-notes.md`, текущую задачу, список изменённых файлов и доступный diff.
2. Убедиться, что этап достаточно значим для отдельного checkpoint; не журналировать каждую мелкую правку.
3. Добавить checkpoint по существующему шаблону и заполнить только подтверждённые факты:
   - решение и его причина;
   - что реально изменилось;
   - реально найденная проблема либо «не выявлено»;
   - только фактически выполненные проверки и их результат;
   - итог этапа.
4. Не записывать chain-of-thought, скрытые рассуждения, секреты или придуманные задним числом события.
5. Сверить запись с diff и результатами команд. Исправить любое утверждение, для которого нет наблюдаемого основания.
