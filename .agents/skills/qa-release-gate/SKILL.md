# QA Release Gate

1. Прочитать `../../../docs/qa-checklist.md` и определить обязательные проверки для текущего этапа.
2. Для каждого пункта использовать ровно один статус:
   - `PASS` — проверка выполнена и прошла;
   - `FAIL` — проверка выполнена и выявила дефект;
   - `NOT RUN` — проверка не выполнялась;
   - `BLOCKED` — проверку невозможно выполнить из-за явно указанной блокировки.
3. Сначала запускать узкие релевантные проверки, затем расширять их соразмерно риску этапа.
4. Не заменять тесты предположениями, не скрывать ошибки и не повышать статус без доказательства.
5. Не выполнять deployment при любом failing mandatory check. `NOT RUN` или `BLOCKED` для обязательной проверки также считать препятствием до явного решения.
6. Выдать итоговый статус gate, доказательства по выполненным проверкам, блокеры и минимальные следующие действия.
